## Liquid Swords 为什么选择 UE5 开发《Samson》

<div class="article-meta">
  <span class="article-category">#UE5</span>
  <span class="article-category">#开放世界</span>
  <span class="article-category">#技术选型</span>
</div>

Liquid Swords 工作室最新开放世界动作游戏《Samson》确认将于4月8日发售。该作采用虚幻引擎5开发，充分利用了 Nanite、Lumen 和 Virtual Shadow Maps 三大核心技术。工作室表示，Nanite 让美术团队能够导入电影级别的精细模型而无需担心性能问题；Lumen 的实时光照系统让关卡设计师可以即时看到最终光照效果，大幅缩短了灯光迭代周期；Virtual Shadow Maps 则解决了大型开放世界场景中的阴影精度与性能矛盾。

### 关键技术：Nanite 让美术更自由

Nanite 的虚拟几何体技术允许开发者直接使用高面数模型（数百万到数十亿面）而无需传统 LOD 流程。对于技术策划而言，这意味着可以在原型阶段就使用接近最终品质的资产进行验证，减少了从灰盒到最终效果的迭代成本。同时，Nanite 自动处理 LOD 切换和裁剪，让团队可以将精力集中在玩法设计上。

### Lumen 实时光照革新工作流

传统的光照烘焙流程需要数小时甚至数天的等待时间，任何调整都代价高昂。Lumen 的全动态光照让设计师可以实时调整光源位置和参数，即时看到效果。这不仅提升了迭代效率，也让"玩法驱动光照设计"成为可能——比如根据游戏节奏动态调整环境氛围，而无需担心烘焙限制。

GitHub 发布 Agentic Workflows 功能，利用大语言模型实现代码库自动化。开发者只需用 Markdown 描述自动化目标和预期结果，AI 智能体就能通过 GitHub Actions 执行这些指令。这为技术策划提供了一种新的自动化思路：用自然语言定义工具链流程。

### 游戏开发中的自动化机会

游戏项目通常包含大量重复性工作：资源导入检查、关卡命名规范验证、性能测试报告生成等。传统做法需要编写复杂的脚本或编辑器插件，维护成本较高。Agentic Workflows 的模式——用自然语言描述需求、让 AI 生成实现——可能降低这类自动化的门槛。

### 技术策划的实用场景

技术策划可以探索以下自动化场景：
- **关卡规范检查**：自动验证命名约定、组件配置、碰撞设置
- **性能回归检测**：自动运行性能测试并对比历史数据
- **文档同步**：代码变更时自动更新相关技术文档
- **构建验证**：自动化冒烟测试，确保关键路径可玩

### 技术策划视角的 UGC 设计

从《我的世界》到《原神》千星奇域，成功的 UGC 平台有一些共同特征：

- **社交发现**：内容需要被看见，推荐算法和社交分享机制至关重要
